# Photo Identify 打包说明

## 目标

将项目打包为 Windows `portable` 目录形式（`PyInstaller onedir`），让其他 Win10/Win11 电脑在具备对应运行时条件下可以直接双击 `photo_identify.exe` 使用。

## 当前结论

本项目**可以打包为 exe**，且当前代码已补充以下打包兼容点：

- 默认数据库路径改为打包目录下的 `data/photo_identify.db`
- 默认缓存路径改为打包目录下的 `data/cache/`
- 运行时自动将 `bin/` 注入 `PATH`，便于调用 `ffmpeg.exe` / `ffprobe.exe`
- GUI 字体支持优先从 `bin/fonts/` 或系统字体目录加载
- 视频转码 helper 会优先从 portable 包内的 `bin/` 使用 `ffmpeg/ffprobe`
- 视频转码 / livp 转换支持优先调用随包 helper exe，回退为源码脚本

## 交付目录建议

最终建议交付如下目录结构：

```text
photo_identify/
├─ photo_identify.exe
├─ _internal/                   # 主 GUI 的 PyInstaller 运行时文件
├─ tools/
│  ├─ video_compression.exe     # helper，onefile
│  └─ lvip_decompression.exe    # helper，onefile
├─ bin/
│  ├─ ffmpeg.exe
│  ├─ ffprobe.exe
│  └─ fonts/
│     └─ seguisym.ttf           # 可选，缺系统字体时附带
├─ data/
│  ├─ photo_identify.db         # 首次运行自动创建
│  └─ cache/                    # 首次运行自动创建
└─ 使用说明.txt
```

## 运行环境要求

目标机器需满足：

1. Windows 10/11 x64
2. 安装 Microsoft Visual C++ Redistributable 2015-2022 x64
3. 若使用 GPU 人脸/向量能力：
   - 已安装 NVIDIA 驱动
   - CUDA 版本与 `onnxruntime-gpu` / 相关依赖兼容
4. 模型不随包分发，需首次启动后在 GUI 中自行配置/下载

## 构建命令（单命令）

### 1. 安装打包依赖

```bat
uv add --dev pyinstaller
```

### 2. 一键构建 portable 包

```bat
uv run python scripts/build_portable.py
```

说明：脚本会自动清空 `dist/`，依次构建主包与 helper，并将 `bin/` 与 helper 复制到 `dist/photo_identify/`。

## 组装 portable 包

使用上面的单命令后，最终产物为：

- `dist/photo_identify/` 目录（**唯一入口**：`photo_identify.exe`）
- helper 放在 `dist/photo_identify/tools/`
- `bin/ffmpeg.exe` / `bin/ffprobe.exe` 位于 `dist/photo_identify/bin/`

## 本机冒烟测试建议

打包后至少验证以下流程：

1. 双击 `photo_identify.exe` 能正常打开 GUI
2. 模型管理页面可打开、保存配置
3. 选择数据库路径后可创建 `data/photo_identify.db`
4. 图片检索页面可正常加载
5. 视频转码按钮可拉起 `video_compression.exe`
6. livp 转换按钮可拉起 `lvip_decompression.exe`
7. `ffmpeg` / `ffprobe` 能被正常调用

## 高风险项

### 1. GPU 依赖

`insightface`、`onnxruntime-gpu`、`opencv` 在不同机器上的 DLL 依赖差异较大，这是最大风险点。若目标机 GPU 环境不一致，可能出现：

- 启动后人脸模块初始化失败
- GPU 推理失败回退到 CPU
- 缺少 CUDA / cuDNN 相关 DLL

### 2. helper exe 依赖

`video_compression.py` 使用 `tqdm`，`lvip_decompression.py` 使用标准库；二者建议独立打包，避免主 GUI 包中再依赖 `sys.executable + .py` 的源码执行方式。

### 3. ffmpeg/ffprobe

当前策略为随包分发，若缺失则视频转码无法使用。

### 4. 字体

若目标机没有 `msyh.ttc` / `seguisym.ttf`，当前代码会降级到默认字体，不影响主功能，但视频缩略图时间角标的显示效果可能变差。

## 干净机器验证清单

建议在一台未安装 Python 的 Windows 电脑验证：

1. 能否直接启动 `photo_identify.exe`
2. 是否提示缺少 VC++ 运行库
3. 是否能创建数据库 / 缓存目录
4. 是否能调用 ffmpeg / ffprobe
5. 是否能成功进行视频转码 / livp 转换
6. GPU 人脸识别是否可用
7. 是否存在 DLL 缺失弹窗

## 现阶段评估结论

**结论：可以打包成可双击运行的 Windows portable exe 程序。**

但为了让“给其他电脑直接双击使用”真正稳定落地，仍建议你至少完成一轮：

- 本机构建
- portable 目录组装
- 干净 Win10/Win11 机器验证

如果只问“能不能”：**能**。
如果问“是否已经达到可稳定分发”：**还需要完成实际构建与目标机验证**。
